System Stabilization & Logic Overhaul Plan

  1. Objective
  Transform the current prototype into a stable, production-ready CLI  application by fixing critical crashes, correcting business logic,
  and improving the developer experience through refactoring.

  ---

  2. Phase 1: Foundation & Stability (The "Bulletproof" Phase)
  Goal: Prevent crashes and ensure data integrity.

  Changes:
   - Centralized Validator: Create Include/Validator.h and
     Source/Validator.cpp to house all rules (e.g., isNonNegative,
     isValidID, isValidDate).
   - Robust Input Handling:
       - Update InputHandler::getInt and getFloat to use try-catch
         blocks for std::stoi and std::stof.
       - Implement a loop that re-prompts the user on invalid input
         instead of crashing.
   - File I/O Safety:
       - Wrap ifstream and ofstream operations in FileHandler with
         basic error checking.
       - Improve the split function in FileHandler.cpp to be more
         resilient to trailing spaces or empty lines.

  ---

  3. Phase 2: Core Business Logic (The "Functional" Phase)
  Goal: Fix the broken rental/sales rules.

  Changes:
   - Sold Status Enforcement:
       - Update SearchEngine to exclude vehicles with status SOLD.
       - Add a check in the Rental process: "If vehicle is SOLD, do
         not allow rental."
   - Return Vehicle Feature:
       - Expose the existing RentalTransaction::setEndTime logic to
         the UI.
       - Add a "Return a Vehicle" option to the Customer/Admin menus.   - Discount & Pricing Logic:
       - Fix the Luxury::calculateCost and Economy::calculateCost
         methods to correctly apply discounts.
       - Ensure zero or negative prices are blocked at the input
         level using the new Validator.
   - Consistent Dates:
       - Replace hardcoded dates with a helper that fetches system
         time or requires a valid DD-MM-YYYY input.

  ---

  4. Phase 3: Architecture & Scalability (The "Clean" Phase)
  Goal: Reduce technical debt and improve maintainability.

  Changes:
   - Decouple main.cpp:
       - Move sub-menu logic (e.g., handleSearch, handleRental) into
         dedicated classes (e.g., MenuHandler or methods within
         Admin/Customer).
       - Aim to reduce main.cpp to under 150 lines.
   - Constants & Configuration:
       - Move all "Magic Numbers" (like the $50-$100 damage fee) into         Include/Constants.h.
       - Update the damage fee to be a percentage of the car's value
         rather than a flat rate.
   - Currency & Localization:
       - Change all currency displays from $ to PKR.

  ---

  5. Phase 4: UI/UX & Polish (The "Professional" Phase)
  Goal: Improve the look and feel for the end-user.

  Changes:
   - Windows CMD Color Fix:
       - Implement a check to enable
         ENABLE_VIRTUAL_TERMINAL_PROCESSING on Windows so ANSI colors         render correctly.
   - Improved Rental Flow:
       - In the "Rent a Vehicle" menu, automatically display the
         available vehicles in the selected category before asking
         for the Vehicle ID.

  ---

  6. Verification & Testing
   - Unit Tests: Create a small test suite to verify Validator logic.   - Manual Regression:
       - Attempt to break the system with invalid inputs (letters in
         price, negative days).
       - Verify that a SOLD car can no longer be rented.
       - Verify that discounts are calculated correctly on the final
         bill.
